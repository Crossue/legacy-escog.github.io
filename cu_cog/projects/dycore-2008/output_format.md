<h1 class="title">Model Output Format</h1>

<div id="cog_post_body">
    <div id="cog_post_body">
        <p>Output conventions</p>
<p>To facilitate the analysis of the results and model intercomparison we ask the modeling group to provide the output data in NetCDF format on a co-located (Arakawa-A type) regular latitude-longitude grid. This does not mean that the model should output directly to NetCDF but the output should be converted to NetCDF format.</p>
<p>The resolution of the chosen latitude -longitude grid should either reflect the actual grid resolution of the model run or should be comparable to the native grid (e.g. comparable to the resolution of a cubed-sphere or icosahedral mesh). Ideally, the horizontal grid should include the Greenwich (zero) meridian and both pole points. In the vertical direction the data should be output on the native grid (full model levels).&nbsp; If variables are staggered in the vertical direction we suggest interpolating them to the full model levels before outputting them. Additional vertical interpolations to fixed pressure levels (e.g. 850 hPa or 250 hPa) will be considered a post-processing step.</p>
<p>The following model variables need to be written to the output NetCDF file. We ask the modeling group to select the upper-case variable names shown below and to use the keywords <em>lat, lon, lev</em> and <em>time</em> for the dimensions. This choice eases the use of standard analysis packages.</p>
<p>The output model variables are:<br /> PHIS (lat,lon)&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp; &nbsp; &nbsp;&nbsp; &nbsp; Surface geopotential (m^2/s^2)<br /> PS (time,lat,lon)&nbsp;&nbsp;&nbsp; &nbsp;&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; Surface Pressure (Pa)<br /> P(time,lev,lat,lon)&nbsp;&nbsp;&nbsp; &nbsp; &nbsp; &nbsp;&nbsp; &nbsp; Pressure at each grid point (Pa), optional if an analytic reconstruction is possible (e.g. for sigma or eta levels)<br /> U (time,lev,lat,lon) &nbsp;&nbsp;&nbsp; &nbsp; &nbsp;&nbsp; &nbsp; Zonal wind (m/s)<br /> V (time,lev,lat,lon) &nbsp;&nbsp;&nbsp; &nbsp;&nbsp; &nbsp; &nbsp; Meridional wind (m/s)<br /> T (time,lev,lat,lon)&nbsp;&nbsp;&nbsp; &nbsp;&nbsp; &nbsp; &nbsp;&nbsp; Temperature (K)<br /> OMEGA (time,lev,lat,lon)&nbsp;&nbsp; Vertical pressure velocity (Pa/s)</p>
<p>If height-based vertical coordinates are used, the vertical velocity might also be output as 'W' in m/s. Note that the notation of the variables above uses the NetCDF-specific order of the dimensions (time,lev,lat,lon). This does not imply that the model needs to work with this data structure. We request the model data with single precision (float) accuracy using 4 bytes per datum.<br /> In addition, the NetCDF file must contain information about the grid and time dimensions, the positions of the horizontal grid and vertical model levels as well as information on the time stamps (the latter are preferred in double precision accuracy, 8 bytes per datum).</p>
<p>An example of an NCAR CAM3 output file on a regular 91&times;144 latitude-longitude grid with grid spacing 2&ordm;&times;2.5&ordm; and 26 hybrid eta-levels is shown below. Note that this output data set also lists the approximate pressure positions of the 26 full levels (lev) and 27 model interface levels (ilev) as well as the hybrid coefficients for the full (hyam, hybm) and half levels (hyai, hybi). The latter can be used in combination with the surface pressure to reconstruct the actual pressure at each grid point. The surface geopotential PHIS only needs to be provided as a 2D data set. The time-dependent data sets PS, U, V, T and OMEGA (not listed) contain 11 instantaneous snapshots in time at day 0, 1, 2, 3, ... 10. In general, shorter output intervals will also be needed (e.g. 3-hourly). The output frequency should be an input parameter for the model run.</p>
